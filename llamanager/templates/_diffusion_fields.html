{# Schema-driven diffusion-profile field renderer. Used by both /ui/diffusion-models (profile editor) and /ui/images (per-request override composer). The macros read the per-engine ProfileField list and emit the right input per kind ("text", "int", "float", "select"). Keep this file the single source of truth for how a profile field looks in the UI; templates should just call ``schema_fields(...)`` and not re-implement the per-kind switch. Each input is named ``{name_prefix}{field.key}`` so the parent form can post all of them at once and the server can demultiplex by stripping the prefix. Defaults render as the input's ``placeholder`` so an unset value shows the engine baseline but still POSTs as empty (the server treats empty as "inherit"). #} {% macro field_input(field, value, name_prefix="field_") %} {% set fname = name_prefix ~ field.key %} {% if field.kind == 'select' %} {% elif field.kind == 'int' %} {% elif field.kind == 'float' %} {% else %} {% endif %} {% endmacro %} {# Render a full block of fields. ``profile`` is the optional pre-fill source: a dict with .fields, as produced by ``_diffusion_models_ctx``. ``name_prefix`` lets callers namespace fields (e.g. the image composer uses "override_" so the server can tell composer overrides apart from a profile-edit save). #} {% macro schema_fields(schema, profile=None, name_prefix="field_") %} {% for f in schema %}
{{ field_input(f, (profile.fields[f.key] if profile and f.key in profile.fields else ''), name_prefix=name_prefix) }} {% if f.help %}{{ f.help }}{% endif %}
{% endfor %} {% endmacro %}
