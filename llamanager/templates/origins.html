Origins

An origin is a named client identity — a script, an app, or a user — that is issued one bearer API key. When multiple origins send requests at the same time, priority (0–100, higher wins) decides who gets the GPU first. Origins marked admin can also call the control plane (start/stop the model, manage other origins, etc.).
