{# ---- GPU for text inference ---- #}
GPU for text inference

Which GPU the text engine runs on. On a machine with more than one GPU (e.g. a discrete card next to an integrated one) llama.cpp may otherwise split the model across both and run far slower — pin it to your fastest card here.

{% if not gpu_devices %}

No GPUs detected. This needs an installed llama.cpp engine with a GPU backend (Vulkan / ROCm / CUDA). Install one from the Setup page, then reload this page.

{% endif %}

Takes effect the next time the engine starts — restart the running model (Dashboard → swap/restart) to apply now. Pinning sets the backend's visible-devices variable (e.g. GGML_VK_VISIBLE_DEVICES) and is stored by device name, so it survives the device-order changes that make a raw index unreliable.

{# ---- Engine behavior ---- #}
Engine behavior

When on, an incoming request can't start the engine or switch models. Requests the currently-loaded model can serve still run; others are rejected until you load the model manually or turn this off.

{# ---- Memory guardrails ---- #}
Memory guardrails

Stop a model from exhausting memory during use. A large context or an accumulating prompt cache can creep past VRAM and RAM until the machine thrashes — these limits catch that before it happens.

{# ---- Conversation retention ---- #}
Conversation retention

How long to keep the prompt and response text shown when you open a request from the dashboard or queue. Set to 0 to never store conversation content — only token counts and timing are kept. Request history (counts, timing, status) is retained separately for up to 90 days regardless.
