Request-level overrides for API clients

API clients pick the model and (optionally) a profile per request via two headers. If the requested pair differs from what's loaded, llamanager hot-swaps automatically.

Model (required to override the default)
X-Llamanager-Model: org/repo-GGUF/Q4_K_M.gguf
Profile (optional; must be bound to that model or be the global default)
X-Llamanager-Profile: fast

The legacy X-Llamanager-Model: profile:<name> shorthand is no longer accepted; the server returns a 400 explaining the new contract.

Each origin's allowed_models setting restricts which models it can request. Set to * on the Origins page to allow any model.
